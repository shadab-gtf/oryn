## SPECIFIC SUBSYSTEM RULES

### ScrollTrigger Documentation Rule
Before implementing ANY ScrollTrigger, document: trigger timing, section start/end, motion intensity, pin duration, overlap logic, section continuity, GPU impact, mobile adaptation, cleanup lifecycle. No random timelines allowed.

### Animation Documentation Rule
Before implementing animations, document: animation purpose, emotional intention, cinematic pacing, easing philosophy, interaction hierarchy, atmospheric layering, masking strategy, transition continuity. Animation must NEVER exist purely for aesthetics.

### Micro-Interaction Documentation Rule
Before implementing interactions, document: environmental response behavior, cursor interaction philosophy, typography response logic, atmospheric motion systems, hover restraint strategy. Micro-interactions must feel subconscious and cinematic.

### Loader + Page Transition Rule
Before implementation, document: initialization timing, loader orchestration, page continuity, transition synchronization, animation activation hierarchy. NO website animation initializes before loader completion.

### Performance-First Rule
Before implementing, document: Lighthouse impact, rendering cost, hydration cost, memory pressure, mobile GPU limitations, SVG complexity, timeline complexity, media delivery strategy. ORYN is cinematic BUT lightweight.
